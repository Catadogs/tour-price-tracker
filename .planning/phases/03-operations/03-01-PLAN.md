### Plan 03-01: Operations hardening

**Task 3.1: Docker healthcheck (OPER-01)**

Add `HEALTHCHECK` directive to `docker/price-monitor/Dockerfile`:

```dockerfile
HEALTHCHECK --interval=30s --timeout=10s --start-period=15s --retries=3 \
  CMD python -c "import sqlite3, os; con = sqlite3.connect(os.environ.get('BG_DB_PATH', '/data/price_monitor.sqlite3')); con.execute('SELECT 1'); con.close()" || exit 1
```

This verifies:
- Python runtime works (no import errors)
- DB volume is mounted and accessible
- SQLite connection succeeds

**Task 3.2: Backup and restore documentation (OPER-02)**

Add a new `## Backup and restore` section to `README.md` covering:

1. **Hot backup** (while container is running):
   ```bash
   docker compose cp bg-price-monitor:/data/price_monitor.sqlite3 ./backup-price_monitor.sqlite3
   ```

2. **Restore**:
   ```bash
   docker compose cp ./backup-price_monitor.sqlite3 bg-price-monitor:/data/price_monitor.sqlite3
   docker compose restart bg-price-monitor
   ```

3. **Alternative with docker compose run** (when container is stopped):
   ```bash
   docker compose run --rm -v $(pwd):/backup bg-price-monitor cp /data/price_monitor.sqlite3 /backup/
   ```

Include a note that backups should be done periodically and the SQLite file is a self-contained, portable database.
