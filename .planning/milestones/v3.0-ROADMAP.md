# Roadmap v3.0: Competition & Discovery

## Phase 1: Cross-Provider Comparison
**Goal**: Бот сравнивает цены одного отеля у Biblio-Globus, Level.Travel и Travelata.

**Depends on**: v2.0 (shipped)
**Requirements**: CMP-01, CMP-02

**Success Criteria**:
1. Отели матчатся по названию между провайдерами (fuzzy matching через difflib, порог 0.6).
2. В Telegram-отчёте появляется блок `🌐 *Сравнение цен*` когда отель найден у 2+ провайдеров.
3. Для каждого отеля: минимальная цена BG (с датой и ночами) + референсные цены конкурентов.
4. Все 72 теста проходят.

## Phase 2: New Arrivals Detection
**Goal**: Бот сообщает о появлении новых дат вылета и новых отелей/номеров в выдаче.

**Depends on**: Phase 1
**Requirements**: NEW-01, NEW-02

**Success Criteria**:
1. Новые даты вылета (отсутствовавшие в предыдущем снепшоте) выделяются отдельным алертом `📅 *Новые даты вылета*`.
2. Новые комбинации отель/номер на существующих датах выделяются алертом `🏩 *Новые отели/номера*`.
3. Алерты не срабатывают на первом запуске (пустой снепшот).
4. Все тесты проходят.

## Phase 3: Container Registry
**Goal**: Docker-образ публикуется в GitHub Container Registry, можно запускать без сборки.

**Depends on**: Phase 2
**Requirements**: GHCR-01, GHCR-02

**Success Criteria**:
1. GitHub Actions CI пушит образ в `ghcr.io/<owner>/bg-price-monitor:latest`.
2. `docker-compose.yml` содержит закомментированный пример с готовым образом.
3. CI проходит (тесты + сборка + пуш).
