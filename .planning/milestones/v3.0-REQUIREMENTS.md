# Requirements: Personal Tour Price Tracker Bot v3.0

**Defined:** 2026-05-04
**Milestone:** v3.0 Competition & Discovery
**Core Value:** Бот должен показывать сравнение цен с конкурентами и предупреждать о появлении новых вариантов.

## v3.0 Requirements

### Cross-Provider Comparison

- [ ] **CMP-01**: Fuzzy-матчинг отелей между провайдерами — если один и тот же отель есть в Biblio-Globus, Level.Travel и Travelata, группировать результаты по названию отеля (через difflib).
- [ ] **CMP-02**: Секция сравнения в отчёте — новый блок `🌐 *Сравнение цен*` с ценами одного отеля у разных провайдеров. Минимальная цена BG (с датами/ночами) + референсные цены Level.Travel/Travelata.

### New Arrivals Detection

- [ ] **NEW-01**: Обнаружение новых дат вылета — если в выдаче появилась дата, которой не было в предыдущем снепшоте, выделить отдельным алертом.
- [ ] **NEW-02**: Обнаружение новых отелей/номеров — если для существующей даты появился новый отель или категория номера, выделить в отчёте.

### Container Registry

- [ ] **GHCR-01**: CI заливает Docker-образ в GitHub Container Registry (ghcr.io) при пуше.
- [ ] **GHCR-02**: docker-compose.yml поддерживает использование готового образа из GHCR вместо локальной сборки.
