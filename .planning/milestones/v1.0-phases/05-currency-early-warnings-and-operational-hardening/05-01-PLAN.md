<objective>
Add daily exchange rate monitoring for USD/RUB and EUR/RUB with configurable alert thresholds. Store observations in SQLite and send preemptive Telegram warnings when significant rate movement suggests tour RUB prices may be recalculated.
</objective>

<execution_context>
### New module: price_monitor/currency.py
- `fetch_exchange_rates(source_url)` — fetches from CBR JSON mirror or flat JSON API
- `format_currency_alert(pair, current, previous, threshold)` — formats alert message
- `run_currency_check(db_path, source_url, threshold)` — orchestrates fetch/store/alert

### Storage: currency_observations table
- `CREATE TABLE currency_observations(id, pair, rate, observed_at)`
- `save_currency_observation()` and `load_currency_observations()` in storage.py

### MonitorConfig: new fields
- `currency_source_url` (default: CBR JSON mirror)
- `currency_alert_threshold_pct` (default: 1.0%)
- `currency_check_hours` (default: 24)

### Main loop integration
- Currency check runs periodically based on `currency_check_hours`
- Alert sent via `send_telegram()` when threshold exceeded
- Failures logged with `logging.exception`

### Tests (tests/test_currency.py)
- `test_fetch_exchange_rates_parses_cbr_style_json`
- `test_fetch_exchange_rates_parses_flat_json`
- `test_format_currency_alert_triggered_on_large_increase`
- `test_format_currency_alert_triggered_on_large_decrease`
- `test_format_currency_alert_suppressed_below_threshold`
- `test_format_currency_alert_returns_none_for_no_previous`
- `test_currency_observations_round_trip`
</execution_context>
